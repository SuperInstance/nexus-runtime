# ADR-030: git-agent as orchestration layer (Cloudflare + Jetson hybrid heartbeat)

## Status
PROPOSED

## Context
NEXUS vessels operate on unreliable maritime networks with intermittent connectivity. A centralized cloud-only orchestration model fails when vessels lose connectivity, potentially leaving them without mission updates or safety alerts. Conversely, a pure edge-only model cannot coordinate multi-vessel fleet operations, share learnings, or provide remote oversight. We need an orchestration architecture that provides fleet-level intelligence when connectivity is available but degrades gracefully to fully autonomous edge operation when it is not, while keeping the git-native coordination paradigm consistent across both environments.

## Decision
Implement **git-agent as a hybrid orchestration layer** with two deployment targets. The Cloudflare-deployed git-agent runs in Workers for fleet-level coordination: receiving telemetry digests, dispatching missions to vessel repos via GitHub Issues/PRs, computing fleet trust aggregates, and managing the mission queue in a central repository. The Jetson-deployed git-agent runs the EdgeHeartbeat loop directly on each vessel, pulling missions from `.agent/next`, executing them through the NEXUS VM, and committing results to `.agent/done`. Both agents share the same NexusBridge API surface. The hybrid heartbeat works by: (1) Edge agent commits telemetry and trust events locally, (2) on connectivity, sync pushes to the vessel's remote on GitHub, (3) Cloudflare git-agent detects new commits via webhooks, (4) Cloud agent processes fleet logic and pushes new missions back. When offline, the edge agent continues autonomous operation with cached mission queue and local trust evaluation.

## Consequences
**Positive:** Fleet coordination survives intermittent connectivity; consistent API across edge and cloud; natural load distribution (cloud handles heavy fleet logic, edge handles real-time control); git push/pull is the only sync primitive needed. **Negative:** Eventual consistency introduces latency for fleet-level decisions; requires webhook infrastructure on GitHub; debugging distributed git workflows adds complexity. **Mitigation:** Emergency protocol bridge bypasses normal sync for RED events; structured commit message prefixes enable grep-based fleet state reconstruction.
